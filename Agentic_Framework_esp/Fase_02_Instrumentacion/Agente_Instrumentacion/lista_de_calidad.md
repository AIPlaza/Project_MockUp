# Lista de Calidad: Instrumento de Recolección de Datos

## Objetivo

Asegurar que el instrumento borrador (`Borrador_Cuestionario.md`) sea válido, confiable y apropiado para la población objetivo antes de ser finalizado y aplicado. El proceso de validación implica múltiples pasos, gestionados por el `Agente_Orquestador`.

---

### Paso 1: Validación de Contenido (Juicio de Expertos)

**Acción:** El `Agente_Orquestador` debe enviar el `Borrador_Cuestionario.md` a 3-5 expertos humanos cuyos perfiles coincidan con los definidos en el `Documento_Diseno_Metodologico.md`. El agente les solicitará que evalúen el instrumento basándose en los siguientes criterios.

- [ ] **Pertinencia:** ¿Los ítems miden realmente la dimensión y los indicadores previstos?
- [ ] **Claridad:** ¿El lenguaje de cada ítem es claro, directo y sin ambigüedades para la población objetivo?
- [ ] **Coherencia:** ¿Existe un flujo lógico y consistencia en todo el instrumento?
- [ ] **Adecuación:** ¿Son las instrucciones, el formato y la longitud apropiados para el contexto en el que se aplicará el instrumento?
- [ ] **Suficiencia:** ¿Hay suficientes ítems para cubrir cada dimensión adecuadamente? (Según la `DISTRIBUCIÓN ÓPTIMA` en el prompt).

**Resultado:** Recopilar la retroalimentación de los expertos. Si se requieren cambios significativos, enviar la retroalimentación al `Agente_Instrumentacion` para generar un borrador revisado. Repetir hasta que los expertos lo aprueben.

---

### Paso 2: Prueba Piloto

**Acción:** Una vez superada la validación de contenido, el `Agente_Orquestador` debe coordinar una prueba piloto como se define en el `Documento_Diseno_Metodologico.md`.

- [ ] **Administrar Instrumento:** Aplicar el cuestionario a una pequeña muestra representativa (5-10 participantes) de la población objetivo.
- [ ] **Recopilar Retroalimentación:**
    - [ ] Recoger datos sobre el tiempo necesario para completar el instrumento.
    - [ ] Entrevistar a los participantes para identificar ítems que fueron confusos, ambiguos o difíciles de responder.
- [ ] **Analizar Datos Piloto:**
    - [ ] Si es cuantitativo, realizar un análisis de confiabilidad preliminar (ej. Alfa de Cronbach). El resultado debe cumplir con el criterio predefinido (ej. α ≥ 0.70).

**Resultado:** Basado en la retroalimentación y el análisis, determinar si se necesitan más revisiones. Si es así, iterar con el `Agente_Instrumentacion`.

---

### Paso 3: Revisión Final

- [ ] **Legibilidad y Formato:** Comprobar que la fuente sea clara y legible, con un espaciado adecuado y una presentación profesional.
- [ ] **Accesibilidad:** Si es digital, asegurar que cumpla con los estándares básicos de accesibilidad.
- [ ] **Sensibilidad Cultural:** Verificación final para asegurar que ningún lenguaje o ejemplo pueda ser malinterpretado o considerado inapropiado por la población objetivo.
- [ ] **Anonimato y Ética:** Confirmar que el encabezado y las secciones de cierre declaren correctamente las garantías éticas (confidencialidad, uso académico exclusivo).

---

### Veredicto de Validación

- [ ] **Aprobado para Finalización:** El instrumento ha superado con éxito todas las etapas de validación. El `Agente_Instrumentacion` ahora puede producir el `Cuestionario_Final.md`.
- [ ] **Rechazado:** El instrumento ha fallado en un paso crítico de validación y requiere una reelaboración significativa. (Proporcionar retroalimentación detallada del paso fallido al `Agente_Orquestador`).
